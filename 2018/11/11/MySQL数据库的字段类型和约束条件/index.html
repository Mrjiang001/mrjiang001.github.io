MySQL数据库的字段类型和约束条件

一.存储引擎

存储引擎是用于根据不同的机制处理不同的数据。

  • innodb:默认存储引擎,

  • myisam: MySQL老版本在用的存储引擎

  • blackhole: 黑洞,存任何数据，都会立即消失，相当于经过队列，数据存入就会被取走

    • memory: 内存引擎(数据存放在内存中,断电即消失,即关闭服务端)

二.创建表完整的语法

总结:

  • 创建表的时候字段名不能重复
  • 最后一个字段不能在末尾加逗号
  • 字段名必须有字段类型与宽度

三.字段类型

  • 确定表结构
  • 字段与字段类型

补充:设置严格模式

在一些低版本的mysql里面,即时输入的数据不符合规格,也会存入进字段里面,所以可以通过设置严格模式来解决这个问题

1)整型

  • 分类:tinyint、smallint、mediumint、int、bigint
  • 应用场景: 存储年龄,等级,id,号码等
  • 典型存储范围

  • 验证整型字段有无符号及范围

2)浮点型:

  • 分类: float、double、 decimal
  • 应用场景: 身高、体重、薪资
  • 三者最大的区别是存储精确度不一样

3)字符类型

  • 分类：varchar 、char

  • 应用场景: 姓名、地址、描述信息

    1)char:定长字符

2)varchar: 不定长

4）日期类型

  • 分类:
    • date: 2019-12-10
    • time: 11:11:11
    • datetime: 2019-12-10 11:11:11
    • year: 2019
    • timestamp:时间戳

表的内容:

mysql> select *from teacher1;

5)枚举和集合类型

分类:

  • 枚举enum 多选一
  • 集合set 多选一 或 多选多
